## 二、本周研发完成事项（核心，量化进度）

### 1. 需求开发任务

* **【FAST-LIO 坐标系标准化改造】**：完成 SLAM 输出与 ROS 2/Nav2 标准坐标系的全面对齐，将 FAST-LIO 全系输出（TF、里程计、点云、路径）由私有 `camera_init`/`body` 坐标系改造为标准 `odom`/`base_link`，并将 `/Odometry` 重映射至标准 `/odom` 话题，打通"模式 A（高精直连 SLAM 定位导航）"链路。
  * **进度**：100% 改造完成，实车导航已跑通。
* **【底盘串口看门狗与到点防漂移重构】**：在底盘驱动 `diuniu_base_node.py` 中引入高精 20Hz 控制看门狗（Watchdog）与主动发包机制。当导航到点（Nav2 停止发布 `/cmd_vel` 超过 0.2 秒）时，看门狗自动触发安全停机，主动且持续向单片机下发 $v=0, 	heta=0$ 锁死指令，根治了小车到点后由于旧指令残留产生徐徐慢爬、越滑越远的硬件漂移缺陷。
  * **进度**：100% 重构完毕，实车验证通过。
* **【手柄遥控与导航通道隔离】**：重定向手柄发布话题至 `/cmd_vel_joy`，在底盘节点内拆分了导航回调与手柄回调。在车辆静止时（$v=0$），手柄的转向控制只会引起前轮的原地转向打角（如方向盘对齐），而驱动轮保持 0 速度完全静止，彻底规避了手柄原地调方向时轮子意外旋转走车的安全风险。
  * **进度**：100% 隔离完毕，实车完美运行。
* **【抑制转向轮抖动与过热防护】**：将 RPP 局部控制器的最小预瞄距离 `min_lookahead_dist` 从过小的 `0.6`m 回调并阻尼收敛至 **`0.9` 米**。前轮低速转弯时的剧烈高频打角扭摆彻底消失，转向电机工作负荷降至正常，电机过热现象根本解决，同时保障了切弯时的前进牵引力连贯输出。
  * **进度**：100% 调参整定完毕。
* **【大拐弯全局路径优化与防卡死】**：禁用 RPP 的原地自转功能（`use_rotate_to_heading: false`），规避了货叉长车身原地旋转大半径盲区刮墙引发的避障锁死。同时将 `global_costmap` 膨胀半径提升至 **`0.55`m**、衰减因子调小至 **`1.5`**，强制全局路径规划在过弯时大幅向弯道外侧“画大弯”绕行，完美解决“车头过了车尾被刮卡住”的内轮差内切问题。
  * **进度**：100% 调试验证完毕。
* **【AMCL 稀疏特征定位优化】**：在 AMCL 配置中将 `max_beams` 从 `60` 调大至 **`120`**。大幅提升了在空旷工厂和长直走廊等稀疏特征场景下的点云匹配与粒子滤波收敛稳定性，减少定位跳变。
  * **进度**：100% 优化完成。

### 2. 测试 / BUG 修复

* **【正前方避障失效 BUG 修复】**：重构 `footprint` 尺寸为 `[[1.60, 0.35], [1.60, -0.35], [-0.25, -0.35], [-0.25, 0.35]]` 并调整过滤器上限至 `x_max = 1.60`，消除了车前 1 米自遮挡过滤盲区，前向避障反应灵敏度完全恢复。
* **【串口占用及僵尸进程死锁清除】**：排查并强杀清除了工控机后台 11 个残留冲突僵尸进程，串口 `/dev/ttyUSB0` 读写异常归零，TF 多源冲突根治，链路恢复纯净（Publisher: 1, Subscriber: 1）。
* **【时钟不同步 SLAM 卡死修复】**：在 `mid360.yaml` 中设置 `time_sync_en: false`，消除了由于未接外部时钟源引起的 1.17 秒时延回环，FAST-LIO 启动稳定性达到 100%。

## 三、下周工作计划

1. **多路导航可靠性压测**：在 Mode A（FAST-LIO 直接定位）和 Mode B（AMCL 传统定位）下运行 2 小时以上的连续导航循环压测，统计到达位置偏差（精度应优于 $\pm 5	ext{cm}$），验证软硬件在长时间运行下的温升与系统负荷。
2. **速度斜坡（Ramp）曲线优化**：配合前轮转向打角速度，在底盘驱动中加入更加平滑的加速度和减速度平滑过渡（S-Curve），避免大角度打角时前轮突然偏转起步造成的整车瞬时机械晃动。
3. **开机一键脚本开发**：编写一键式 shell 启动与监控脚本（提供状态检查和进程守护），实现雷达、FAST-LIO、Chassis Base、Nav2 导航系统的低门槛一键式拉起。
